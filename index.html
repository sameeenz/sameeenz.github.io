Adding EXIF metadata to digitised photos with Linux CLI.

March 23, 2019

Today I got a few films from high school digitised, after they came off the scanner they didn’t have any metadata relating to the date in which they were actually taken.

I wanted to add a bit of metadata so they would be sensibly arranged in my photo content management system – I had to guess a bit. Within the nearest year was good enough.

I found that a really nice tool exists for this task called ‘exiftool’ created by Phil Harvey. Thanks Phil, cracker tool. On Ubuntu it can be installed by:

sudo apt-get install exiftool

You can find the documentation here: https://www.sno.phy.queensu.ca/~phil/exiftool/

Here is a one-liner I found to convert all .jpegs in a folder to have a date of 2007. It will also overwrite the existing image, so exercise caution!

exiftool '-datetimeoriginal=2007:03:01 12:00:00' -overwrite_original *.jpg

Finding dimensions off a scale drawing with GIMP

January 7, 2019

Today I was trying to figure out whether or not a bicycle pannier rack would fit my bicycle. I’m thinking of cycling to work a bit in an effort to curb last years 1274.8L of petrol, and to improve my fitness a bit. Recently half of the Kamo to Whangarei cycleway opened, and I feel a bit safer going to work by bicycle now.

I found an nice pannier rack, fair price, alloy, has a spring-a-ma-thing to hold things on. There was even a scale drawing on the manufacturer’s website to help me…

…but unfortunately the dimension I was after was not listed.

GIMP has a tool where you can measure angles and lengths (Shift-M), or the drawing compass icon on the toolbar. This was used first to compare two known dimensions against each other to find a common ratio to convert pixels of the image to the dimension in mm. This step was also important to verify that the technical drawing was to scale, a cheap sanity test before proceeding further.

With the known ratio of pixels:mm conversion I dragged out the length with the measurement tool to find the minimum length the rear dropout could be from the top mounting point while still having a level top. The minimum distance is 210±20mm, and the measurement worked out to about 180mm, heaps of room!

With any luck and a fair breeze it will turn out as nice as this one.

Edit 12/1/2019: After posting this article to Twitter, I got this excellent reply from @jigpu (see below) explaining how the ratio of pixels/mm can be changed in the “Image -> Print Size” menu function. After that is set then you can just use the measurement tool to measure ‘mm’ (adjusted in the bottom left, as pictured below). A great tip to save time and a lot of mathematics if you’re taking a number of measurements. Thanks @jigpu!

"Pro Tip: After working out the pixels per mm as described, set it in the “Image -> Print Size” menu and then change the units drop-down inside the status bar from “px” to “mm”. Now the measure tool will directly report real-world size — no further math required! Inches too 🙂" — Jason Gerecke (@jigpu) January 8, 2019
